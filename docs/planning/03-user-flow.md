## 전체 네비게이션 흐름

### 1.1 Mermaid 전체 흐름도

```
graph TD
    A["메인 대시보드<br/>/"] -->|SQL 실행기 클릭| B["SQL 실행기<br/>/playground"]
    A -->|개념 설명 클릭| C["시각적 개념 설명<br/>/concepts"]
    A -->|용어 사전 클릭| D["SQLD 용어 사전<br/>/glossary"]

    B -->|상단 네비 클릭| A
    B -->|상단 네비 클릭| C
    B -->|상단 네비 클릭| D
    B -->|'개념 보기' 클릭| C

    C -->|상단 네비 클릭| A
    C -->|상단 네비 클릭| B
    C -->|상단 네비 클릭| D
    C -->|'실습해보기' 클릭| B

    D -->|상단 네비 클릭| A
    D -->|상단 네비 클릭| B
    D -->|상단 네비 클릭| C
    D -->|'실습 버튼' 클릭| B
```

### 1.2 네비게이션 규칙

| 이동 경로 | 트리거 | 설명 |
|---------|--------|------|
| 어디서나 → 대시보드 | 로고/홈 버튼 클릭 | 즉시 대시보드로 돌아감 |
| 어디서나 → SQL 실행기 | 상단 "SQL 실행기" 메뉴 | 이전 쿼리 히스토리는 유지 |
| 어디서나 → 개념 설명 | 상단 "개념 설명" 메뉴 | 마지막 본 개념은 기억 |
| 어디서나 → 용어 사전 | 상단 "용어 사전" 메뉴 | 검색어는 유지 |
| 개념 설명 → SQL 실행기 | 개념의 "실습해보기" 버튼 | 해당 예제 SQL 자동 입력 |
| 용어 사전 → SQL 실행기 | 용어의 "실습 버튼" | 해당 용어의 SQL 예제 자동 입력 |

## 화면별 사용자 시나리오

### 2.1 메인 대시보드 시나리오

#### 시나리오 A: 첫 방문

```
사용자 진입
    ↓
[페이지 로드]
- 헤더: SQLD Visual Lab 로고 + 메뉴
- 메인 콘텐츠: "안녕하세요, SQLD Visual Lab입니다"
- 3개 기능 카드 표시:
  [ SQL 실행기 ]  [ 개념 설명 ]  [ 용어 사전 ]
- 각 카드에 설명 + "시작하기" 버튼
- 푸터: 정보 + 링크
    ↓
사용자가 "SQL 실행기" 카드의 "시작하기" 버튼 클릭
    ↓
/playground로 네비게이션
```

#### 시나리오 B: 재방문

```
사용자 진입
    ↓
[페이지 로드]
- 메인 대시보드 표시
- LocalStorage에서 "마지막 방문 시간" 로드
- (선택) "안녕하세요, {이름}님" 인사 표시
    ↓
사용자가 "용어 사전" 메뉴 클릭
    ↓
/glossary로 네비게이션
```

### 2.2 SQL 실행기 시나리오

#### 시나리오 A: 샘플 데이터로 시작

```
페이지 진입: /playground
    ↓
[UI 로드]
- 상단: SQL 에디터 + 샘플 데이터 드롭다운
- 하단: 비어있는 결과 테이블
    ↓
사용자가 "샘플 데이터 선택" 드롭다운에서 "사원 (EMPLOYEES)" 선택
    ↓
[sql.js 초기화]
- EMPLOYEES, DEPARTMENTS 등 샘플 테이블 로드
- 데이터 확인 메시지 표시 (토스트)
    ↓
[에디터 자동 채우기]
- SQL 에디터에 기본 쿼리 자동 입력:
  SELECT * FROM employees LIMIT 10
    ↓
사용자가 "실행" 버튼 클릭
    ↓
[쿼리 실행]
- sql.js에서 쿼리 실행
- 결과 행 10개 반환
    ↓
[결과 표시]
- 하단 테이블에 10행 표시
- 컬럼: id, name, dept_id, salary, hire_date
- 페이지네이션: "10 / 10행"
```

#### 시나리오 B: 자유 쿼리 작성

```
페이지 진입: /playground
    ↓
[데이터 미리 로드]
- 기본 샘플 데이터 자동 로드
    ↓
사용자가 SQL 에디터 클릭 → 직접 쿼리 입력
- 입력: SELECT name, salary FROM employees WHERE salary > 50000
    ↓
사용자가 "실행" 버튼 또는 Ctrl+Enter 단축키 입력
    ↓
[쿼리 실행]
- sql.js에서 쿼리 실행
- 결과 행 3개 반환
    ↓
[결과 표시]
- 테이블에 3행 표시 (name, salary 컬럼만)
    ↓
[히스토리 저장]
- 이 쿼리를 LocalStorage에 저장
- "최근 쿼리 10개" 목록에 추가
    ↓
사용자가 "히스토리" 버튼 클릭 → 이전 쿼리 재실행
```

#### 시나리오 C: 에러 처리

```
사용자가 잘못된 SQL 입력
- 입력: SELECT * FORM employees  (오타: FORM)
    ↓
사용자가 "실행" 버튼 클릭
    ↓
[쿼리 실행]
- sql.js 에러 캐치
- 에러 메시지: "Syntax error near 'FORM'"
    ↓
[에러 표시]
- 결과 영역에 빨간 박스로 에러 메시지 표시
- 에러 해결 힌트 제공 (선택사항)
    ↓
사용자가 SQL 수정 → "실행" 버튼 재클릭
```

#### 시나리오 D: 테이블 생성 및 조작

```
사용자가 CREATE TABLE 쿼리 입력
- 입력: CREATE TABLE test (id INT, name VARCHAR(50))
    ↓
사용자가 "실행" 버튼 클릭
    ↓
[테이블 생성]
- sql.js에서 테이블 생성 완료
- 메시지: "Table 'test' created"
    ↓
사용자가 INSERT 쿼리 입력
- 입력: INSERT INTO test VALUES (1, 'Alice')
    ↓
사용자가 "실행" 버튼 클릭
    ↓
[데이터 삽입]
- sql.js에서 행 삽입 완료
    ↓
사용자가 SELECT 쿼리 입력
- 입력: SELECT * FROM test
    ↓
사용자가 "실행" 버튼 클릭
    ↓
[데이터 조회]
- 결과 테이블에 1행 표시
```

### 2.3 시각적 개념 설명 시나리오

#### 시나리오 A: 개념 탐색

```
페이지 진입: /concepts
    ↓
[UI 로드]
- 좌측: 개념 목록
  [ SELECT ] [ WHERE ] [ INNER JOIN ] [ LEFT JOIN ] ...
- 우측: 비어있는 설명 영역
    ↓
사용자가 좌측 "INNER JOIN" 클릭
    ↓
[우측 영역 업데이트]
- SVG 벤 다이어그램 렌더링
  (두 개의 원이 겹치는 모양, 교집합 강조)
- 텍스트 설명 표시:
  "INNER JOIN은 두 테이블의 공통 행만 반환합니다."
- SQL 예제 표시:
  SELECT e.name, d.dept_name
  FROM employees e
  INNER JOIN departments d ON e.dept_id = d.id
- "실습해보기" 버튼
    ↓
사용자가 "실습해보기" 버튼 클릭
    ↓
[SQL 실행기로 이동]
- /playground로 네비게이션
- SQL 에디터에 위의 예제 SQL 자동 입력
- 샘플 데이터 자동 로드
```

#### 시나리오 B: 개념 간 비교

```
사용자가 "LEFT JOIN" 클릭
    ↓
[우측 영역 업데이트]
- 새로운 SVG 다이어그램 (LEFT JOIN 시각화)
- 왼쪽 원이 강조, 오른쪽 겹침 부분은 옅음
- "LEFT JOIN은 왼쪽 테이블의 모든 행과..."
    ↓
사용자가 "RIGHT JOIN" 클릭
    ↓
[우측 영역 업데이트]
- 오른쪽 원이 강조
    ↓
사용자가 좌측 목록에서 다른 개념 클릭
    ↓
[스크롤 및 하이라이트]
- 좌측에서 현재 선택된 개념이 강조 표시
```

## 에러 시나리오

### 3.1 SQL 실행기 에러

#### E1: 문법 오류

```
사용자 입력: SELECT * FORM employees

실행 결과:
├─ 에러 타입: SyntaxError
├─ 에러 메시지: "Syntax error near 'FORM'"
├─ 사용자에게 보여줄 메시지: "SQL 문법 오류입니다. 'FORM'을 확인해주세요."
├─ 해결 방법 (선택):
│  - 유사한 올바른 명령어 제안: "혹시 SELECT를 찾으셨나요?"
└─ 버튼: [코드 수정]
```

#### E2: 테이블 없음

```
사용자 입력: SELECT * FROM non_existent_table

실행 결과:
├─ 에러 타입: RuntimeError
├─ 에러 메시지: "no such table: non_existent_table"
├─ 사용자에게 보여줄 메시지: "테이블을 찾을 수 없습니다. 샘플 데이터를 로드했는지 확인해주세요."
└─ 버튼: [샘플 데이터 다시 로드]
```

#### E3: 컬럼 없음

```
사용자 입력: SELECT invalid_column FROM employees

실행 결과:
├─ 에러 타입: RuntimeError
├─ 에러 메시지: "no such column: invalid_column"
├─ 사용자에게 보여줄 메시지: "컬럼을 찾을 수 없습니다. 존재하는 컬럼명을 확인해주세요."
├─ 팁: "employees 테이블의 컬럼: id, name, dept_id, salary, hire_date"
└─ 버튼: [컬럼 목록 보기]
```

#### E4: 권한 오류 (DDL 제한)

```
사용자 입력: DROP TABLE employees

실행 결과:
├─ 사용자에게 보여줄 메시지: "기본 샘플 테이블은 삭제할 수 없습니다."
└─ 팁: "새로운 테이블을 직접 생성하고 삭제할 수 있습니다."
```

## 상세 플로우 다이어그램

### 4.1 사용자 의사결정 트리

```
사용자 진입
    │
    ├─ 첫 방문?
    │   ├─ YES → 대시보드 표시 + 사용 설명
    │   └─ NO → 마지막 페이지로 이동 (LocalStorage)
    │
    ├─ 사용자가 선택:
    │   │
    │   ├─ SQL 연습하고 싶음?
    │   │   └─ /playground로 이동
    │   │       ├─ 샘플 데이터 로드?
    │   │       │   └─ 도움말 표시
    │   │       ├─ 쿼리 입력
    │   │       ├─ 실행 클릭
    │   │       ├─ 성공?
    │   │       │   ├─ YES → 결과 표시 + 히스토리 저장
    │   │       │   └─ NO → 에러 표시 + 수정 요청
    │   │       └─ 반복...
    │   │
    │   ├─ JOIN, GROUP BY 개념을 배우고 싶음?
    │   │   └─ /concepts로 이동
    │   │       ├─ 개념 선택
    │   │       ├─ 설명 + 다이어그램 표시
    │   │       ├─ 실습하기 클릭?
    │   │       │   └─ /playground로 이동 (쿼리 자동 입력)
    │   │       └─ 다른 개념 선택 (반복)
    │   │
    │   └─ 용어의 뜻을 찾고 싶음?
    │       └─ /glossary로 이동
    │           ├─ 검색 또는 카테고리 선택
    │           ├─ 용어 찾기
```
