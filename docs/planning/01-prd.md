## 프로젝트 개요

### 1.1 제품명
**SQLD Visual Lab** - SQL 개념 시각화 학습 도구

### 1.2 한 줄 설명
SQLD 시험을 준비하는 학생들이 SQL 개념을 시각적으로 이해하고 브라우저에서 직접 실습할 수 있는 웹 기반 학습 도구

### 1.3 목적
- **교육적 목표**: SQLD 시험 범위의 SQL 개념을 직관적으로 이해하도록 지원
- **실용적 목표**: 책의 예제를 직접 실행하고 실험하면서 학습
- **접근성 목표**: 개발 환경 설치 없이 브라우저만으로 즉시 시작

### 1.4 대상 사용자
- **주 대상**: SQLD(SQL 개발자 자격증) 시험 준비 중인 학생
- **사용자 레벨**: L1 (개발 경험 없는 초보자)
- **예상 사용자 특성**:
  - SQL을 처음 배우거나 기초가 약함
  - SQLD 시험 범위에 나오는 개념만 학습하고 싶음
  - 손으로 직접 SQL을 쳐보며 학습하기를 원함
  - 개발 환경 설정이 복잡하다고 느낌

### 1.5 시장 기회
- SQLD 시험 응시자: 연 5-10만 명
- 기존 SQLD 강의는 영상 중심이며 실습 환경이 제한적
- 무료 웹 기반 학습 도구 부족

## 문제 정의

### 2.1 현재 학생들의 고충
| 고충 | 설명 |
|------|------|
| **학습의 추상성** | SQL 개념(JOIN, GROUP BY 등)이 어떻게 동작하는지 이해하기 어려움 |
| **실습 환경 설정의 복잡성** | 데이터베이스 설치/설정이 번거로워 바로 시작하지 못함 |
| **피드백 부재** | 책의 예제를 읽기만 하고 직접 실행해볼 수 없음 |
| **용어 이해의 부족** | SQLD 시험에 나오는 용어를 한 번에 찾기 어려움 |
| **실습 데이터 부족** | 공식 문제집의 예제 데이터가 제한적 |

### 2.2 경쟁 상황
- **기존 온라인 IDE** (e.g., SQLFiddle, db-fiddle): 범용 도구, SQLD 특화 X
- **SQLD 강의 플랫폼**: 영상은 풍부하나 실습 도구가 부족
- **오프라인 학원**: 높은 비용, 시간 제약

### 2.3 우리의 차별화
1. **SQLD 시험 특화**: 시험에 나오는 개념과 데이터만 정제
2. **시각화 중심**: 벤 다이어그램, 테이블 변환 과정을 그림으로 표현
3. **0초 시작**: 로그인, 설치 없이 바로 사용
4. **통합 경험**: 개념 설명 → 실습 → 용어 검색을 한 도구에서

## 핵심 기능

### 3.1 기능 1: SQL 실행기 (Playground)

#### 개요
브라우저에서 SQL을 직접 입력하고 실행하면 결과가 테이블로 표시되는 환경

#### 사용 시나리오
1. 학생이 SQLD 교재의 SQL 예제를 본다
2. "SQL 실행기" 메뉴 클릭
3. 좌측 드롭다운에서 "사원 테이블"을 선택 → 자동으로 샘플 데이터 로드
4. SQL 에디터에 예제를 입력: `SELECT * FROM employees WHERE salary > 50000`
5. "실행" 버튼 클릭
6. 하단에 결과 테이블 표시

#### 주요 기능
- **상하 분할 레이아웃**
  - 상단: SQL 입력 에디터 + 샘플 데이터 선택 드롭다운
  - 하단: 결과 테이블 (정렬, 페이지네이션 지원)
- **샘플 데이터 자동 로드**
  - 사원(EMPLOYEES), 부서(DEPARTMENTS), 급여(SALARIES), 주문(ORDERS) 등
  - 각 테이블별 10-50개 행으로 구성
  - SQLD 시험의 예제 데이터와 동일 구조
- **자유로운 실험**
  - 학생이 CREATE TABLE로 새 테이블 생성 가능
  - INSERT, UPDATE, DELETE도 지원
  - 이전 쿼리 히스토리 제공
- **에러 표시**
  - SQL 문법 오류 시 에러 메시지 표시
  - 초보자 친화적 에러 설명 제공

#### 기술 구현
- SQLite (sql.js 사용, 브라우저 메모리에서 실행)
- 코드 에디터: Monaco Editor (VS Code 기반)
- 결과 테이블: React Table 컴포넌트

### 3.2 기능 2: 시각적 개념 설명 (Concepts)

#### 개요
JOIN, GROUP BY, 서브쿼리, 정규화 등 SQLD 핵심 개념을 그림과 애니메이션으로 설명

#### 사용 시나리오
1. 학생이 "개념 설명" 메뉴 클릭
2. 좌측에 개념 목록 표시: [INNER JOIN] [LEFT JOIN] [GROUP BY] [서브쿼리] ...
3. "INNER JOIN" 클릭
4. 우측에 표시:
   - **시각화**: 두 개 테이블의 벤 다이어그램, 교집합 강조
   - **설명**: "INNER JOIN은 두 테이블의 공통된 행만 반환합니다"
   - **SQL 예제**:
     ```sql
     SELECT e.name, d.dept_name
     FROM employees e
     INNER JOIN departments d ON e.dept_id = d.id
     ```
   - **미니 실행기**: 예제를 바로 실행해볼 수 있는 실행 버튼

#### 주요 기능
- **개념별 시각화** (SVG + CSS 애니메이션)
  - JOIN 종류별 벤 다이어그램 (INNER, LEFT, RIGHT, FULL OUTER)
  - GROUP BY의 행 묶음 과정을 단계별로 표시
  - 정규화 단계별 테이블 분해 과정
  - 서브쿼리의 실행 순서 시각화
- **텍스트 설명**
  - 초보자 눈높이의 쉬운 설명
  - "언제 사용하는가?" 실무 예제
- **실습 연계**
  - 각 개념마다 "실습해보기" 버튼
  - 클릭 시 SQL 실행기로 이동, 해당 예제 SQL 자동 입력

#### 다룰 개념 목록 (SQLD 빈출)
1. SELECT, WHERE, ORDER BY
2. INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN
3. GROUP BY, HAVING, 집계 함수 (COUNT, SUM, AVG 등)
4. 서브쿼리 (SELECT, FROM, WHERE 절 서브쿼리)
5. UNION, UNION ALL, INTERSECT, EXCEPT
6. 정규화 (1NF, 2NF, 3NF)
7. 인덱스, 뷰, 트랜잭션 (TCL 개념)

#### 기술 구현
- SVG 기반 다이어그램 렌더링
- Framer Motion으로 애니메이션 처리
- Markdown 기반 설명 텍스트

### 3.3 기능 3: SQLD 용어 사전 (Glossary)

#### 개요
SQLD 시험에 자주 나오는 용어를 검색하고, 카테고리별로 탐색하며, 각 용어마다 쉬운 설명과 SQL 예제를 제공

#### 사용 시나리오
1. 학생이 "용어 사전" 메뉴 클릭
2. 상단 검색창에서 "서브쿼리" 검색 → 즉시 결과 표시
3. 카테고리 탭 선택: [DDL] [DML] [TCL] [정규화] [그 외]
4. "DML" 탭 클릭 → DDL 명령어들 필터링 표시
5. "INSERT" 카드 클릭:
   - **용어**: INSERT
   - **정의**: "테이블에 새로운 행을 추가하는 DML 명령어"
   - **용도**: "데이터 생성 작업에 사용됨. 트랜잭션 대상."
   - **SQL 예제**:
     ```sql
     INSERT INTO employees (name, salary)
     VALUES ('John', 50000)
     ```
   - **버튼 1**: "예제 보기" → 예제 코드 복사
   - **버튼 2**: "실습하기" → SQL 실행기로 이동, 예제 자동 입력

#### 주요 기능
- **빠른 검색**
  - 상단 검색창: 입력 즉시 필터링
  - 자동완성 제안
- **카테고리 필터링**
  - DDL (CREATE, ALTER, DROP 등): 10개 용어
  - DML (SELECT, INSERT, UPDATE, DELETE 등): 12개 용어
  - TCL (COMMIT, ROLLBACK, SAVEPOINT): 5개 용어
  - 정규화 (1NF, 2NF, 3NF, BCNF): 8개 용어
  - 그 외 (인덱스, 뷰, 제약조건 등): 15개 용어
- **용어 카드 구성**
  - 용어명, 분류, 정의, 용도, SQL 예제, 관련 용어 링크
- **SQL 실행기 연계**
  - 각 용어의 "실습하기" 버튼 클릭 시 예제 SQL이 자동 입력됨
- **모바일 친화적**
  - 카드 기반 레이아웃으로 스크롤 가능

#### 용어 데이터 관리
- JSON 파일: `public/data/glossary.json`
- 구조:
  ```json
  {
    "terms": [
      {
        "id": "term-insert",
        "name": "INSERT",
        "category": "DML",
        "definition": "테이블에 새로운 행을 추가하는 DML 명령어",
        "usage": "데이터 생성 작업에 사용됨. 트랜잭션 대상.",
        "example": "INSERT INTO employees (name, salary) VALUES ('John', 50000)",
        "relatedTerms": ["DML", "트랜잭션"]
      }
    ]
  }
  ```

#### 기술 구현
- React 검색 필터링 (useState, useMemo)
- Fuse.js로 모호 검색 (타이핑 실수 대응)
- JSON 정적 데이터 로드

## 사용자 스토리

### 4.1 메인 사용자 스토리

| 우선순위 | 스토리 | 예상 이득 |
|----------|--------|----------|
| 1️⃣ | **SQLD 학생으로서**, 책에 나온 SQL 예제를 바로 실행해보고 싶다 | 학습의 피드백 루프 단축 |
| 2️⃣ | **SQLD 학생으로서**, JOIN이 어떻게 동작하는지 그림으로 보고 싶다 | 개념 이해도 향상 |
| 3️⃣ | **SQLD 학생으로서**, '서브쿼리'같은 용어의 뜻을 쉽게 찾아보고 싶다 | 학습 시간 단축 |
| 4️⃣ | **SQLD 학생으로서**, SQL을 치면 결과가 바로 나와서 실감이 나게 공부하고 싶다 | 학습 동기 부여 |
| 5️⃣ | **SQLD 학생으로서**, 개념 설명을 보다가 바로 실습해보고 싶다 | 학습 경험의 연속성 |
| 6️⃣ | **SQLD 학생으로서**, SQLD 시험 범위에 맞는 샘플 데이터로 연습하고 싶다 | 시험 적중률 향상 |

### 4.2 수용 기준 (Acceptance Criteria)

**AS-1: SQL 예제 즉시 실행**
- GIVEN: 학생이 SQL 실행기 페이지에 있음
- WHEN: SQL 에디터에 `SELECT * FROM employees` 입력하고 실행 버튼 클릭
- THEN: 1초 이내에 결과 테이블이 표시됨

**AS-2: 개념 시각화 이해**
- GIVEN: 학생이 "INNER JOIN" 개념 설명을 봄
- WHEN: 벤 다이어그램이 표시됨
- THEN: 초보자도 이해할 수 있는 수준의 명확한 다이어그램

**AS-3: 용어 검색 성능**
- GIVEN: 학생이 용어 사전에 있음
- WHEN: 검색창에 "JO"로 시작하는 글자 입력
- THEN: 0.1초 이내에 관련 용어 필터링

**AS-4: 샘플 데이터 SQLD 준거성**
- GIVEN: SQL 실행기에서 샘플 데이터 선택
- WHEN: EMPLOYEES 테이블 로드
- THEN: SQLD 교재의 예제 데이터와 동일한 스키마/데이터

## 제약 조건

### 6.1 기술 제약
| 제약 | 이유 | 영향 |
|------|------|------|
| **서버 없음** | 배포 비용 최소화 | 모든 SQL이 브라우저 메모리에서 실행 (sql.js) |
| **정적 데이터** | Vercel 무료 호스팅 | JSON 파일로 데이터 관리, 실시간 동기화 X |
| **SQLite만 지원** | sql.js의 한계 | MySQL, PostgreSQL 특화 기능 제공 불가 |
| **로컬 저장만 가능** | 서버 없음 | 사용자 작업 저장 = 브라우저 LocalStorage (브라우저별 독립) |

### 6.2 비즈니스 제약
| 제약 | 이유 | 영향 |
|------|------|------|
| **SQLD 범위 내** | 타겟 사용자 명확화 | 고급 SQL, NoSQL 다루지 않음 |
| **무료 배포** | 수익화 계획 없음 (MVP) | Vercel 무료 티어 제한 (빌드 시간 45분/월, 대역폭 100GB) |
| **1인 팀 가정** | 초기 단계 | UI/UX 매력도보다 기능 정확도 우선 |

### 6.3 사용자 제약
| 제약 | 이유 | 영향 |
|------|------|------|
| **PC/노트북 우선** | 초기 타겟 학습 환경 | 모바일(스마트폰)은 2차 최적화 대상 |
| **한국어만** | 초기 대상 사용자가 국내 SQLD 응시자 | 다국어 지원 로드맵 없음 |
| **로그인 불필요** | 진입 장벽 최소화 | 개인화/맞춤형 학습은 불가 |
